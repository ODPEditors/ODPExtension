Category Browser

A full-screen menu that allows navigate, filter or find for categories in the entire directory.

  • Objective
    • Allows quick access to any category in the directory.
    • Most important: Help when you want to apply an action to a category.
  • Usages:
    • Basically has two usages:
      • [ action to category ] For when you previously selected and action (example: edit a category description ) and wants to find the category to apply that action.
      • [ category to action ] For when  you want to find a category to then apply an action (example: edit a category description ).
    • Invocations methods
      • In the case of [category to action ] you need first to find a category to later select an action to apply to that category.
        • Example:
          • By clicking the "/" toolbarbutton the "Category Browser" popups if you click any category the default action is copy to clipboard the category path. If you want to apply another action to the category just right click the hovered category to see a menu of predefined actions ( The "From category" menu, help about this menu later )
      • In the case of [action to category] you need first to select an action to later find for the category to apply that action.
        • Example:
          • Let's say that we want to edit the description of a category. Open a new blank tab. Now double click with the Right button of your mouse ( inside the blank page ), the "From category" menu with the predefined actions will popup, then select "Edit Description", that will open the "Category Browser" menu asking you for the category were you want to edit the description.

  • Explanation of the menu items
    • Textbox filter finder
      • Usages:
        • Basically has two usages:
          • Filter the content of the category browser menu.
          • Append searches of the categories.txt database to the category browser menu.
        • Filtering the content
          • This textbox allows you to type a search term and the menu will filter the contents to show you only categories that match the search term.
            • Example
              • This search term "development -computer -bus, development arts -computer" will show only the categories that contain "development" but will hide these containing "computer" and "bus"  OR will show these containing "development" AND "arts" but will hide these containing "computer".
        • Appending searches
          • Another usage of this textbox is to append searches from the categories.txt database, this is useful for when the category that you are trying to find is no present yet in the menu.
            • Example
              • ( assuming that you already have processed the "categories.txt" database, clicking "update now" on "Preferences -> Advanced" ) you will see a button "search" and a listbox with some of the top categories in groups. Then you can enter some search term and press "search" to do a deep search on the database.
                • This search will detect if it is a REGEXP and use that if applicable.
    • Reset menu
      • This is simply a button that "resets" the menu to the default configuration, for example to clean the menu of searches made on the categories.txt database, or too free resources if to many categories were added to the menu.
    • Top
      • A shortcut to some top categories.
    • This browsing session categories
      • Holds all the categories that you previously navigated on this browser session. ( cleaned on browser shutdown )
    • Categories opened
      • It will add a shortcut to the every category that is opened in your browser tabs.
    • Clipboard categories
      • Will list all the categories that you have on your clipboard
    • Searches on the categories.txt
      • Foreach search on the categories.txt the menu will list the categories and the amount of categories found.
    • Locked categories
      • Because you maybe access to some categories more frequently than others, this menu allows you to set some "locked" categories, that never will be removed, configurable via Preferences -> Category Browser.
    • Category history
      • Every time you open or edit a category, this menu saves the category on a database, so the next time you open this menu the categories are there. Also it tries to add some relevance to the menu, for example giving priority to categories that you edit most frequently.
  • Navigation
    • Another cool feature of this menu is that allows you to navigate the directory.
    • When you hover a category X the extension will read the public or editor pages and parse the content to dynamically build a submenu that shows the subcategories of the hovered category. If the category starts with Test, the server to read is the editors server, this is because Test is private ( non public accessible ). If you already downloaded and processed the categories.txt database the extension will look for subcategories there if applicable.
    • When the navigation reach the limit ( when the category doesn't' have subcategories ) a slash is added to the category path to indicate that.
    • Currently this feature only navigates subcategories, not @links, altlangs or related categories.
  • Applying an action
    • Another cool feature is that every category in this menu has a context menu. You can right click any category to immediately apply an action to the selected category.
  • Source of the data
    • This feature has several sources to fill the menu with categories.
      • Categories that are currently opened in the browser
      • Categories that were opened in this browser session
      • Categories that you keep on the clipboard
      • Categories searches from the categories.txt database
      • Locked categories that are set on the extension preferences
      • Every time you visit or edit a category the category is saved in a SQLite database. ( category history )
      • Editors server ( editors.dmoz.org )
      • Public server ( www.dmoz.org )
      • Categories imported from your past browsing history
  • Importing categories from your past browsing history
    • You have the option to import to the database the categories that you were navigating/editing in the past. See: Preferences -> Category Browser -> Import categories from my browsing history button. This is a cool feature because: fills the menu with relevant categories to you. It also give to most edited/accessed categories priority on the menu, it's probably that your favourite categories will remain on top of the menu.
  • Resources
    • By doing lots of searches the menu increase by an appreciable amount of categories. To keep the resources low this menu automatically short the number of items to 100 every time the popup is closed.
